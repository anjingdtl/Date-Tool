### 排错经验（重要）

## 真 LLM 接入与三大 bug 修复（17:3x）
世恒哥质疑分析没用真 LLM、截图出现 `__EMPTY_4`（SheetJS 空列占位符）。逐个解决：

### Bug 1：`.env.local` 变量名与代码不匹配 → LLM 全程跑 Mock
- 用户填了 `LLM_API_KEY` / `LLM_BASE_URL` / `LLM_MODEL`，但 `lib/config.ts` 读的是 `OPENAI_API_KEY` / `OPENAI_BASE_URL` / `OPENAI_MODEL`，key 为空 → `config.llm.enabled=false` → 全程 Mock（连 narrative 都是 Mock 写死的「你好，世恒哥～」前缀，曾误判为真 LLM）。
- 修复：config.ts 兼容两套变量名（`OPENAI_*` 优先，回退 `LLM_*`）。`.env.local` 保持 LLM_* 写法即可。

### Bug 2：Excel 合并单元格/隐藏列 → 字段名变成 `__EMPTY_4`
- SheetJS 把空列自动命名为 `__EMPTY` / `__EMPTY_1`...，被当成真实字段进分析。
- 修复：parse.ts 增加 `stripEmptyColumns`（过滤行内 `__EMPTY*` 键）+ buildColumns 过滤 `^__EMPTY(_\d+)?$` 列名。

### Bug 3：MiniMax M3（推理模型）把 `<think>...</think>` 思考过程塞进输出
- 非流式 `chatJSON`：JSON 前裹 `<think>`，`stripJson` 已加剥离（正则 + 鲁棒截取 `{...}`）。
- 流式 `streamChat`：推理内容作为 `text` token 随流输出，前端会原样展示。修复：新增 `filterThink`（跨 chunk 状态机，buffer 暂存被截断标签头），只回调正式文本；`full` 也不含思考内容。
