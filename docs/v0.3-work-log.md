## 阶段 2：DataContext 和隐私 — ✅ 完成（2026-07-16）

### 完成内容
- **`lib/semantic/detect-sensitive.ts`**：敏感字段检测（字段名关键词 + identifier 角色 + 值模式手机/邮箱/身份证/银行卡）+ 稳定掩码（保留长度、首尾可见、同值同掩码的一致映射，纯函数无 `Math.random`/`crypto`）。
- **`lib/semantic/build-data-context.ts`**：构建 `DataContext`（SPEC 9）：
  - 确定性哈希（djb2）+ PRNG（mulberry32），种子 = `hashStr(datasetId + rowsHash)`，保证同数据同采样。
  - 采样：头 8 / 中 6 / 尾 6 / 随机 12，按行 index 去重，上限 40；`boundaryRows`（数值 min/max/p25/p75）、`anomalyCandidateRows`（IQR，≤10）。
  - 列上下文：复用 `computeNumericStats`/`computeCategoryStats`，只含客观数据（`detectedType`/`heuristicHints`/`numericStats`/`topValues`），不写业务结论。
  - 脱敏：发给 LLM 的样本经 `maskRow`；数值统计用原值（不因掩码改变）。
  - `tokenBudget`（4 字符/token 估算）+ `omittedSections`（关闭行样本时记录）。
  - 提示注入防护：单元格值仅作数据保留，不改变结构（SPEC 9.5）。
- **测试**：`tests/sensitive-mask.test.ts` + `tests/data-context.test.ts`（覆盖规格 24.2 矩阵：小表全量、大表 ≤40、头中尾覆盖、稳定性、不同 hash、脱敏、高基数裁剪、token 截断、注入防护、不含业务结论）。

### 设计决策
- 不复制统计逻辑：直接复用 `lib/analysis/statistics`、`lib/analysis/profile`（SPEC 13.1「不得复制两套相同计算」）。
- `truncated` 语义 = 原始行数 > 40（数据多于能发送的样本量），清晰反映采样信息有损。
- `DataContext.quality` 在旧数据缺失时兜底构造；`generatedAt` 随构建时间（采样/统计仍稳定）。
- 修复潜在栈溢出：日期极值用循环求 min/max，不用 `Math.min(...bigArray)`。

## 阶段 5：公式引擎与工具注册表 — ✅ 完成（2026-07-16）

### 完成内容
- **`lib/executor/formula-engine.ts`**（SPEC 11）：`validateFormulaAST`（深度≤8 / 节点≤40 / 字段存在）+ `evalFormula`（递归 AST 求值，缺失值与除零返回 null，禁 eval/new Function）。
- **`lib/executor/result-hash.ts`**：sha256 + 任务规范化 JSON + `hashTaskInput`（rowsHash + understandingHash + executorVersion + 任务）+ `hashResult`。
- **`lib/executor/task-cache.ts`**：进程内 LRU（500 条），命中复用（SPEC 14.3）。
- **`lib/executor/operators/_shared.ts`**：filter（10 种 operator）/ sort / 数值提取 / 7 种聚合（含 median/last）/ evidence 工厂 / 字段存在性校验。
- **9 个 P0 操作符**：profile / aggregate / timeseries / compare / distribution / ranking / ratio / growth / anomaly。
  - timeseries 支持 day/week/month/quarter/year，默认按 measureBehavior（stock→last / rate·score→avg / 其余→sum）。
  - ratio 执行安全公式（除零 whenZero 策略）；anomaly 复用 `lib/analysis/outliers` 的 IQR + 自实现 z-score。
  - 每个工具生成 Evidence（method 映射 EvidenceMethod），不复制统计逻辑（复用 `computeNumericStats`/`computeCategoryStats`/`detectOutliers`）。
- **`lib/executor/registry.ts`**：注册 9 操作符 + `dispatchTask`（inputHash→缓存→校验→执行→填 inputHash→写缓存，失败返回 failed 不抛错）+ `validateTask`/`getTool`/`listOperators`。
- **测试**（SPEC 24.5 矩阵）：formula-engine / tool-registry / executor-aggregate / timeseries / ratio / growth / anomaly + 共享 fixture。

### 设计决策
- correlation / pivot 留待阶段 6（与计划执行器一起）。
- timeseries 的 `bucketOf`/`toDateStr` 导出供 growth 复用，避免复制分桶逻辑（SPEC 13.1）。
- operator 不算 inputHash（无 rowsHash/understandingHash 上下文），由 registry.dispatch 统一算并回填 result + evidence。
- 测试修正：IQR/zscore 对密集异常失效是统计特性，测试数据保持异常占比 <25% 以验证真实检出。
