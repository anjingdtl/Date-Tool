## 阶段 1：类型和 Schema — ✅ 完成（2026-07-16）

### 完成内容
- **`lib/types.ts` 扩展**：新增 v0.3 全部纯类型契约（按规格第 8-17 节）：
  - 状态机：`UnderstandingStatus` / `AnalysisSessionStatus` / `RevisionStatus` / `AnalysisMode`。
  - 语义层：`DatasetKind` / `TableShape` / `SemanticFieldRole` / `MeasureBehavior` / `SemanticSubRole` / `SemanticAggregation` / `AnalysisOperator`，及 `FieldUnderstanding` / `FieldRelationship` / `DerivedMetricSuggestion` / `UnderstandingAmbiguity` / `DatasetUnderstanding`。
  - 公式：递归 AST `FormulaExpression`（SPEC 11.2，禁 eval）。
  - DataContext：`WorkbookContext` / `SheetContext` / `HeaderCandidate` / `ColumnDataContext` / `DataContext`。
  - 计划：`TaskFilter` / `TaskSort` / `AnalysisTask` / `DashboardItemPlan` / `DashboardPlan` / `AnalysisPlan`。
  - 执行：`TaskExecutionResult` / `PlanExecutionResult` / `ToolDefinition` / `ToolExecutionContext`。
  - Review：`ReviewFinding` / `ChartReviewDecision` / `AnalysisReview`。
  - Patch：`AnalysisPlanPatch`。
  - Session/Revision：`AnalysisSession` / `AnalysisRevision`。
  - 最终结果：`FinalAnalysisResult extends AnalysisResult`（`analysisMode` / `reviewStatus` / `planSummary`）。
  - Evidence 扩展：`EvidenceMethod`（8→15）+ 任务关联可选字段。
- **5 个 Zod schema**（运行时校验，依赖层级无环：formula → understanding/analysis-plan → plan-patch → analysis-review）：
  - `lib/schemas/formula.ts`（递归 `z.lazy` + `discriminatedUnion`）。
  - `lib/schemas/understanding.ts`、`analysis-plan.ts`（复用 `FieldFormatSchema`）、`plan-patch.ts`、`analysis-review.ts`。
- **`components/InsightPanel.tsx`**：`METHOD_LABEL` 补全 7 个新 EvidenceMethod 标签（类型联动修复）。
- **5 个测试文件**（正反例）：formula / understanding / analysis-plan / analysis-review / plan-patch schema。

### 设计决策
- 纯 TS 类型集中在 `lib/types.ts`（单一类型契约）；运行时校验集中在 `lib/schemas/*`，二者结构通过测试对齐，避免双重维护与循环依赖。
- `FormulaExpressionSchema` 用 `discriminatedUnion`（每个 op 单项），结构层禁动态执行；深度/节点上限留待阶段 5 formula-engine 校验。
- 不用 `any` 绕过核心协议（规格阶段 1 明令）。
